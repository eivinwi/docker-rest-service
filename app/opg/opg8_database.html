Docker volumer og databaser

Det finnes ferdige images for databaser, og Docker Hub innneholder versjoner for de fleste populære dialekter som MySQL, PostgreSQL og MongoDB etc. Om ingen av imagene på Hub passer ens spesifikasjoner, eller man av andre grunner ikke ønsker å bruke de offisielle imagene er det også fint mulig å lage dem selv. To eksempler på dette er lenket under.

For større systemer kan det lønne seg å benytte mer avanserte løsninger for data-volume managment i docker som f.eks Flocker.

a)

Bruk søkekommandoen for å finne det offisielle MySQL-imaget, og kjør en container av dette imaget. Aksesser det ved å starte bash i containeren med kommandoen 'docker exec -it <navn eller id> bash', og derfra bruke MySQL-kommandoer. Opprett en database, og fyll inn litt innhold.

MySQL kommandoer:
  • 'mysql -p' starter mysql i terminalen
  • CREATE DATBASE <db-navn>
  • SHOW DATABASES;
  • USE <db-navn>
  • CREATE TABLE <table-navn>

b)

Stop containeren og start den igjen, bruk samme kommando som i forrige oppgave og se at databasen fortsatt inneholder alle de samme dataene.
