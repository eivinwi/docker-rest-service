Docker Machine

Som vi så i forrige oppgave er docker-apper fine og portable, men å lagre til fil og flytte de rundt manuelt er hverken smidig eller praktisk, og skalerer dårlig til større systemer. Docker-Machine kan brukes for å forenkle distribusjonen av docker-images til eksterne servere, og også til å automatisere denne prosessen.

a)

Bruk Docker-Machine for å lage en machine som kan kjøre docker-containere lokalt i Virtualbox.

Tips: en oversikt over tilgjengelige docker-machines (både lokalt og remote) kan finnes med kommandoen 'docker-machine ls'. 'docker info <machine-navn> viser informasjon om maskinen.

b)

Docker-machine støtter de fleste store hosting providers, og man trenger faktisk kun én kommando i docker-machine for å automatisk starte opp en server i skyen, installere og konfigurere docker på serveren og sette opp SSH med keypair! Bruk docker-machines 'create' kommando for å starte opp en Amazon AWS-server med utdelt API-key. Last deretter opp imaget laget i forrige oppgave.

c)

Bruk docker-machines innebygde ssh-kommando til å logge inn på serveren. Logget inn på docker-machinen kan man bruke samme docker-kommandoer som lokalt, for å verifisere at docker-containeren kjører! Prøv f.eks 'docker images' og 'docker ps'.

Annen info
