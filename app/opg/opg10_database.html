Docker volumer og databaser

Databaser har i utgangspunktet det samme problemet som logg-filer når de kjøres i docker-containere, nemlig at de blir borte når containeren stoppes. Heldigvis finnes det spesiallagde images for databaser, og Docker Hub innneholder versjoner for de fleste populære dialekter som MySQL, PostgreSQL og MongoDB etc. Om ingen av imagene på Hub passer ens bruk, eller man av andre grunner ikke ønsker å bruke de offisielle imagene er det også fint mulig å lage dem selv. To eksempler på dette er lenket under.

For større systemer kan man også benytte seg av mer avanserte løsninger for data-volume managment i docker som f.eks Flocker.
