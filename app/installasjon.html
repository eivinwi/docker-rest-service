Installasjon og kjøring av Docker

Hvordan docker installeres er forskjellig avhengig av operativsystem. Installasjon og bruk av Docker er noe enklere og krever færre steg på Linux, men man kan også fint bruke Mac eller Windows. Lenker til de separate installasjonsinnstruksene finnes i tabellen under.

Operativsystem Informasjon
Linux Docker
Linux DockerMachine
Enkel, rett frem installasjon. Separat installasjon for Docker-machine (kreves fra og med oppgave 5).
Krever 64-bits versjon av Linux.
Ikke installer direkte fra package-managere som Apt/Yum/Pacman da docker konstant oppdateres, og versjonene derfra ofte blir utdaterte.
Virtualbox må også installeres for den siste delen av workshoppen.
Tips: for å slippe å kjøre Docker som root kan du bruke kommadoen 'sudo usermod -aG docker <brukernavn>'
Mac
Anbefaler i denne omgang den eldre "Docker Toolbox"-versjonen, da det er den vi selv har testet tidligere.
Relativt enkel installasjon.
Krever minst OS X 10.8 Mountain Lion.
Krever Virtualbox (Dersom du har Virtualbox installert fra før huker du ikke av på installering av Virtualbox under installasjonen)
(Spesielt interesserte kan som alternativ heller prøve den nye native-implementasjonen)
Windows
VIKTIG: Default-installasjon på Windows er nå en native-implementasjon som bruker noe som kalles Hyper-V. Denne implementasjonen har dog store utfordringer med flere av kjernekonseptene i Docker, og det anbefales derfor at man heller installerer "Docker Toolbox"-versjonen, som baserer seg på Virtualbox!! (lenken til venstre peker på Docker Toolbox)
Litt flere steg enn på Linux/Mac, men selve installasjonen er relativt grei.
Krever minst Windows 7.
Krever 64-bits Windows.
Krever at Virtualization er aktivert i BIOS. (Win8+: kan sjekkes i Oppgavebehandling -> Ytelse)
Krever Virtualbox (Dersom du har Virtualbox installert fra før huker du ikke av på installering av Virtualbox under installasjonen)
Etter installasjon: for å kjøre Docker-kommandoer brukes nylig installerte Docker Quickstart Terminal.
Merk: Docker programmene i seg selv bruker litt diskplass (opptill 400MB for Mac/Windows), og i tillegg brukes det masse plass på å opprette Docker images underveis. Det er derfor anbefalt å ha minst 20 gigabyte med ledig plass på maskinen, for å slippe å slette ting underveis.
For å rydde opp ting laget med Docker etter at workshoppen kan man bruke kommandoene som er forklart her. Merk at noen av disse fjerner alle images/containere laget av Docker, så vær sikker på at du vil fjerne alt før du eventuelt kjører disse. Hvis du velger å avinstallere, så kan det også være kjekt å slette alt under C:\Users\<brukernavn>\.docker eller /var/lib/docker.

Annen nødvendig programvare som forventes installert

  • Java 8 SDK. (Trenger ikke nødvendigvis væore Oracle).
  • Maven 3
  • En fritt valgt SSH-client (OpenSSH, Putty eller lignende).
  • Programvare for filoverføring, som støtter SCP eller FTP.
Merk: OpenSSH følger med i Docker Toolbox, støtter ssh/scp, og kan brukes fra Docker Quickstart Terminal!
