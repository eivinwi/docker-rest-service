Konsepter

Docker Engine er kjernen av Docker. Engine lar deg bygge isolerte beholdere som inneholder programmet ditt, eller hva som helst du har lyst til å kjøre. Dette gjør den ved å kjøre en veldig nedstrippet og spesialisert versjon av linux.

Docker Machine er et verktøy for deployment og automatisering av docker.

Docker Swarm er en måte å bygge clustere på.

Et Image er en read-only template som kan brukes til å lage Containers. Et eksempel på et Image kan inneholde en versjon av Ubuntu, en installasjon av Apache, og en web-applikasjon.

En Container er en instanst av et image. En container er en helt isolert applikasjonsplattform, som inneholder programmer som skal kjøres, og alle nødvendige dependencies.

Docker Engine: kommandoer

Kommando Effekt
docker imagesviser alle tilgjengelige images (som er bygget eller nedlastet)
docker psviser alle kjørende docker containere
docker build -t <name> <dir>leser og bygger Dockerfile fra spesifisert directory
docker run <image> <params....>kjører spesifisert docker-image med parametere
docker inspect <imagenavn / containerid>viser nyttig informasjon om image/container
docker rmi -f <imagenavn / containerid>fjerner spesifisert image
